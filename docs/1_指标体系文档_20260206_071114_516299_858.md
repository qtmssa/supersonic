# 指标体系文档

## 1. 数据库与范围
- 数据库：`postgresql://10.0.12.252:5438/metricsys`（用户 metricsys，密码省略）
- 业务范围：AdventureWorks 销售、渠道、财务与运营指标体系
- 数据量概览：FactInternetSales≈60,398 行，FactResellerSales≈60,855 行，FactProductInventory≈776,286 行，FactFinance≈39,409 行，FactCurrencyRate≈14,264 行，FactCallCenter≈120 行，FactSalesQuota≈163 行，DimCustomer≈18,484 行，DimProduct≈605 行，DimDate≈3,652 行。

## 2. 主题域（Domain）
| 域名 | 描述 | 覆盖表/模型 | 主要使用人群 |
| --- | --- | --- | --- |
| 销售域 | 互联网直销与经销商渠道销售分析，含订单、促销、客户与区域 | FactInternetSales、FactResellerSales、FactInternetSalesReason、DimCustomer、DimProduct、DimSalesTerritory、DimPromotion、DimCurrency、DimDate、DimGeography、DimReseller、DimSalesReason | 销售、市场、增长运营 |

## 4. 语义模型（Model）

### 销售域

#### 模型：互联网销售事实 / `fact_internet_sales`
- 来源：表 `public.FactInternetSales`
- 粒度：`SalesOrderNumber` + `SalesOrderLineNumber`
- 时间分区：`OrderDateKey` 对应 DimDate.DateKey（默认），`DueDateKey`、`ShipDateKey` 可选
- 关联：`ProductKey → DimProduct.ProductKey`；`CustomerKey → DimCustomer.CustomerKey`；`SalesTerritoryKey → DimSalesTerritory.SalesTerritoryKey`；`PromotionKey → DimPromotion.PromotionKey`；`CurrencyKey → DimCurrency.CurrencyKey`；日期键 → DimDate.DateKey；地区通过 `Customer.GeographyKey → DimGeography.GeographyKey`
- 默认过滤：无

维度  
| 名称 | bizName | 类型/格式 | 说明 |
| --- | --- | --- | --- |
| 订单日期 | order_date | 时间 | OrderDateKey→DimDate.FullDateAlternateKey |
| 发货日期 | ship_date | 时间 | ShipDateKey→DimDate |
| 产品 | product | 枚举 | DimProduct，含品类/子类 |
| 品类 | product_category | 枚举 | DimProductCategory/DimProductSubcategory |
| 客户 | customer | 枚举 | DimCustomer，包含姓名/邮箱/收入等 |

| 促销 | promotion | 枚举 | DimPromotion |
| 币种 | currency | 枚举 | DimCurrency |

## 5. 指标（含衍生）
| 名称 | bizName | 类型 | 定义（agg/expr） | 维度下钻 | 过滤/口径说明 | 数据格式 |
| --- | --- | --- | --- | --- | --- | --- |
| 销售额 | sales_amount | MEASURE | sum(FactInternetSales.SalesAmount) | 日期、产品、客户、区域、促销、币种 | 订单行聚合 | currency |
| 渠道销售额 | reseller_sales_amount | MEASURE | sum(FactResellerSales.SalesAmount) | 日期、产品、经销商、区域、币种 | 订单行聚合 | currency |
| 订单数 | orders | METRIC | count(distinct FactInternetSales.SalesOrderNumber) | 日期、产品、客户、区域 | 去重订单 | number |
| 客单价 | aov | METRIC | sales_amount / NULLIF(orders,0) | 同上 | 防除零 | currency |
| 毛利 | gross_profit | MEASURE | sum(SalesAmount - TotalProductCost) | 日期、产品、区域 | 不含税运费 | currency |
| 毛利率 | gross_margin_rate | METRIC | gross_profit / NULLIF(sales_amount,0) | 日期、产品、区域 | 防除零 | percent |
| 折扣率 | discount_rate | METRIC | sum(DiscountAmount) / NULLIF(sum(SalesAmount + DiscountAmount),0) | 日期、产品、促销 | 含折扣后口径 | percent |
| 渠道毛利 | reseller_gross_profit | MEASURE | sum(FactResellerSales.SalesAmount - FactResellerSales.TotalProductCost) | 日期、产品、经销商 |  | currency |
| 渠道毛利率 | reseller_margin_rate | METRIC | reseller_gross_profit / NULLIF(reseller_sales_amount,0) | 日期、产品、经销商 | 防除零 | percent |
| 配额金额 | quota_amount | MEASURE | sum(FactSalesQuota.SalesAmountQuota) | 年/季度、员工、区域 | 预算配额 | currency |
| 配额达成率 | quota_achievement | METRIC | reseller_sales_amount / NULLIF(quota_amount,0) | 年/季度、员工、区域 | 以渠道销售额对比配额 | percent |
| 配额差额 | quota_gap | METRIC | reseller_sales_amount - quota_amount | 年/季度、员工、区域 | 正数为超额 | currency |
| 财务金额 | finance_amount | MEASURE | sum(FactFinance.Amount) | 日期、科目、部门、组织、场景 | 记账金额 | currency |
| 平均汇率 | avg_rate | MEASURE | avg(FactCurrencyRate.AverageRate) | 日期、币种 |  | number |
| 汇率收盘价 | eod_rate | MEASURE | avg(FactCurrencyRate.EndOfDayRate) | 日期、币种 |  | number |
| 入库量 | units_in | MEASURE | sum(FactProductInventory.UnitsIn) | 日期、产品 |  | number |
| 出库量 | units_out | MEASURE | sum(FactProductInventory.UnitsOut) | 日期、产品 |  | number |
| 库存余额 | units_balance | MEASURE | max(FactProductInventory.UnitsBalance) | 日期、产品 | 建议取日末值 | number |
| 库存周转 | inventory_turnover | METRIC | units_out / NULLIF((units_balance),0) | 日期、产品 | 简化口径，可改用期间平均库存 | number |

## 8. 校验与风险
- 元数据未暴露主键/外键，需在建模时手工指定粒度与关联键，避免笛卡尔积。
- 时间字段多为整数键或 time 类型，需在 Supersonic 中映射为日期（DimDate.FullDateAlternateKey）并设默认时间字段。
- 币种转换：销售额为本币，跨币种分析需结合 FactCurrencyRate 做换算，明确目标币种与换算口径。
- 成本口径：FactInternetSales/FactResellerSales 仅有 TotalProductCost，是否含税/运费需业务确认；毛利率计算未扣税运费，可按需要调整。
- 库存余额字段为快照/余额，未提供仓库维度；如需多仓分析需补充仓库信息或外部表。
- FactProductInventory 的 MovementDate 为文本，建议转为日期格式再作为默认时间。
- 数据新鲜度未标识，需确认 ETL 周期；大部分维度/事实允许全量聚合，注意大表 FactProductInventory 量级较大（≈77 万行）。
